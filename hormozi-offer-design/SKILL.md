# Hormozi Offer Design — $100M Offers Framework

## Overview

An offer is not a product. An offer is the complete package of value, terms, bonuses, guarantees, and framing you present to a buyer. The same product with a better offer makes dramatically more money.

**The goal:** Make an offer so good people feel stupid saying no.

---

## The Value Equation

```
Value = (Dream Outcome × Perceived Likelihood of Achievement)
        ÷ (Time Delay × Effort & Sacrifice)
```

Increase value by:
- Making the dream outcome bigger / more specific
- Increasing how likely they believe it'll work for them
- Reducing how long it takes to get results
- Reducing how hard they have to work

To make an irresistible offer: maximize the top, minimize the bottom.

---

## Step 1: Define the Dream Outcome

Write the exact end state the customer wants — in their words, not yours.

**Bad:** "Lose weight"
**Good:** "Fit into the dress you wore at your wedding — in 90 days"

**Bad:** "Grow your business"
**Good:** "Add $30K/month in recurring revenue without working weekends"

Be specific. Name the emotional component. The more vivid the outcome, the more people buy.

---

## Step 2: List Every Problem (Exhaustive)

For every step between the customer's current state and the dream outcome, list every obstacle:

- **Value doubt:** "Is this actually worth it for me?"
- **Likelihood doubt:** "Will this work for my specific situation?"
- **Effort:** "This looks too complicated / time-consuming"
- **Time:** "I won't see results fast enough"

Go exhaustive. Aim for 20+ problems. One unaddressed problem = one reason to not buy.

---

## Step 3: Turn Problems Into Solutions

Flip each problem into a solution component of your offer:

| Problem | Solution (offer component) |
|---|---|
| "I don't know if it'll work for me" | Personalized audit call before starting |
| "Setup takes too long" | Done-for-you onboarding in 48 hours |
| "What if I don't get results?" | 100% money-back guarantee |
| "I'm not technical enough" | Live support + video walkthroughs |
| "I don't have time" | 30-minute weekly sessions only |

This list becomes your offer's components.

---

## Step 4: Choose Delivery Vehicle for Each Solution

For each solution, decide how to deliver it:

| Dimension | Options |
|---|---|
| Effort from customer | DFY (done-for-you) / DWY (done-with-you) / DIY |
| Format | Live / recorded / written / tools |
| Timing | Instant / scheduled / on-demand / async |
| Personalization | Custom / templated / automated |

**10x / 1/10th test:**
- 10x resources: what's the best possible delivery?
- 1/10th resources: what's the minimum viable delivery that still gets the outcome?

The 1/10th version is often your MVP. The 10x version is your premium tier.

---

## Step 5: Add Guarantees (Risk Reversal)

The #1 adoption blocker is risk. Design it out.

| Customer fear | Guarantee design |
|---|---|
| "What if it doesn't work?" | Money-back guarantee |
| "What if I'm not satisfied?" | No-questions-asked refund |
| "What if I do all the work and still fail?" | Conditional guarantee (keep working until it works) |
| "What if I can't figure it out?" | Done-for-you guarantee |

**Hormozi's rule:** If you believe in your product, the guarantee costs you almost nothing. If you don't believe in it, fix the product.

**Conditional guarantee format:**
> "If you complete [specific actions] and don't get [specific result], we'll [give money back / keep working for free]."

Conditions filter for serious buyers. Serious buyers complete the conditions and get results. Win-win.

---

## Step 6: Name the Offer (M-A-G-I-C Formula)

| Letter | Element | Examples |
|---|---|---|
| **M** | Reason why / hook | "Automated," "Instant," "Zero-config," "Guaranteed" |
| **A** | Avatar / who it's for | "for coaches," "for e-commerce brands," "for first-time founders" |
| **G** | Goal / outcome | "first client," "5x revenue," "lose 20 lbs" |
| **I** | Time interval | "in 90 days," "in 30 minutes," "by Friday" |
| **C** | Container / format | "System," "Program," "Blueprint," "Workshop" |

**Bad:** "Business Coaching Program"
**Good:** "The 90-Day Client Acquisition System for Solo Consultants"

---

## Step 7: Stack the Bonuses

After the core offer, add bonuses that each address one remaining obstacle:

- Each bonus should be independently valuable (assign a dollar value)
- Each bonus removes one specific fear or friction point
- Reveal bonuses one at a time, building excitement
- "Oh, and you also get…" repeated 3–5 times

**Bonus stack format:**
> "You get [core offer — $X value]. PLUS [bonus 1 — $Y value] that handles [specific problem]. PLUS [bonus 2 — $Z value] that handles [next problem]. Total value: $[sum]. Your investment: $[price]."

---

## Offer Checklist

```
[ ] Dream outcome written in customer's language (specific, vivid, emotional)
[ ] 20+ problems identified across all 4 dimensions
[ ] Every problem has a corresponding solution/component
[ ] Delivery vehicle chosen for each component
[ ] Guarantee designed to reverse the biggest risk
[ ] Bonuses target remaining friction points
[ ] Each bonus independently priced
[ ] Total value anchor clearly stated
[ ] Offer named using M-A-G-I-C formula
[ ] Offer presented so customer compares value to price, not price to nothing
```

---

## Common Mistakes

| Mistake | Fix |
|---|---|
| Selling the product, not the outcome | Lead with dream outcome; mechanism is secondary |
| No guarantee → high perceived risk | Add a guarantee; it costs almost nothing if your product works |
| Unpriced bonuses → zero anchor value | Price every bonus; unpriced = worthless |
| Generic offer → "who's this for?" | Name the avatar explicitly in the offer |
| Weak name → forgettable | Apply M-A-G-I-C; test on someone unfamiliar with your business |
